# AGH 国际医疗系统 Demo 规格

## 概念与愿景

一个用于给客户展示医疗旅游平台完整解决方案的静态演示页面。通过模拟一位东南亚肿瘤患者的完整就诊旅程，让客户直观看到每个角色的工作界面和系统价值。风格：专业医疗科技感，温暖可信赖。

## 设计语言

- **美学方向**: 医疗科技风 — 蓝白主调，圆角卡片，清晰的信息层级
- **色彩方案**:
  - Primary: #2563EB (科技蓝)
  - Secondary: #0F172A (深墨)
  - Accent: #10B981 (健康绿)
  - Background: #F8FAFC (浅灰白)
  - Card: #FFFFFF
  - Text: #1E293B
  - Muted: #64748B
- **字体**: Inter (Google Fonts), 系统默认
- **动效**: 淡入淡出过渡，卡片hover轻微上浮

## 全链路流程

```
[广告获客] → [患者提交] → [运营采集] → [医生评审] → [医院预约] → [术后随访]
   ↓             ↓            ↓            ↓            ↓            ↓
 患者端       患者端       运营端       评审端       医院端       健康管家端
```

## 五端 Demo 设计

### 1. 患者端（患者视角）
- 落地页：看到Facebook广告后扫码/点击进入
- 病情提交表单：基本信息、病情描述、图片上传
- 方案查看：显示医生给的手术方案、医院、医生
- 术后随访：复查提醒、用药提醒、应急入口

### 2. 运营端（马来28人团队）
- 患者列表：所有患者状态看板
- 新建患者：标准化信息采集表单
- 病历资料：患者上传的所有资料查看
- 提交评审：整理好资料后提交给医生
- 跟踪状态：评审结果、手术安排、归国进度

### 3. 评审端（4位中国医生）
- 待评审列表：按优先级排列的病例
- 评审详情：查看完整病历、打分（病情复杂度、紧急度）
- 出方案：推荐医院、推荐医生、手术方案、预估费用
- 已评审列表：历史评审记录

### 4. 医院端（国内合作医院）
- 预约接收：显示待确认的手术预约
- 档期管理：可预约日期设置
- 手术确认：确认手术安排、反馈病床情况
-术后同步：患者归国后随访数据的同步

### 5. 健康管家端（归国后服务）
- 患者随访计划：归国后各阶段随访节点
- 执行记录：每次随访的结果录入
- 异常预警：复查结果异常、用药问题提示
- 应急响应：7x24应急入口、当前处理中案例

## 技术实现
- 纯静态 HTML + Tailwind CSS (CDN)
- 单页应用，使用 JS 控制 Tab 切换
- 模拟真实数据（一位患者的完整就诊旅程）
- 每个端有独立的导航 Tab
